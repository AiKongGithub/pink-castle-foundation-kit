## SLIDE 4: ตรวจโครงสร้างเนื้อหาจาก Content Builder
```
╔══════════════════════════════════════════╗
║   ด้านที่ 1: โครงสร้างเนื้อหา (20 คะแนน)║
╚══════════════════════════════════════════╝
```

**โครงสร้างที่ถูกต้อง:**
```
┌─────────────────────────────────┐
│  TITLE — ดึงดูด มี Keyword     │
├─────────────────────────────────┤
│  INTRODUCTION — บอกว่าเกี่ยว   │
│  กับอะไร ผู้อ่านจะได้อะไร      │
├─────────────────────────────────┤
│  H2: หัวข้อที่ 1               │
│    H3: หัวข้อย่อย              │
│    เนื้อหา + Visual Break      │
├─────────────────────────────────┤
│  H2: หัวข้อที่ 2               │
│    เนื้อหา + รูปภาพ            │
├─────────────────────────────────┤
│  CONCLUSION — สรุป + ทิศทาง    │
└─────────────────────────────────┘
```

- พบปัญหา: ไม่มี Introduction ที่ดี, ไม่มี Conclusion
- การจัดรูปแบบ: ต้องมี Visual Break ทุก 3-4 บรรทัด, ใช้ Bullet/Bold/รูปภาพ

## SLIDE 5: ตรวจ SEO On-page (จุดตกคะแนนมากที่สุด)
```
╔══════════════════════════════════════════╗
║   ด้านที่ 2: SEO On-page (25 คะแนน)    ║
╚══════════════════════════════════════════╝
```

| องค์ประกอบ | เกณฑ์ |
|-----------|------|
| Title Tag | มี Keyword หลัก, ไม่เกิน 60 ตัวอักษร |
| Meta Description | มี Keyword, สรุปเนื้อหา, ไม่เกิน 160 ตัวอักษร, มี CTA |
| URL | สั้น, สื่อความหมาย, มี Keyword, ใช้ขีดกลาง (-) |
| Heading Hierarchy | H1 (1 ตัว) → H2 (หัวข้อหลัก) → H3 (หัวข้อย่อย) |
| Keyword Placement | อยู่ใน Title, Meta, URL, H1, Paragraph แรก, H2 |

**ข้อผิดพลาดร้ายแรง**: ใช้ Bold แทน Heading — Google ไม่รู้จักว่าเป็นหัวข้อ!

## SLIDE 7: ตรวจ Internal Linking
```
╔══════════════════════════════════════════╗
║   ด้านที่ 4: Internal Linking (15 คะแนน)║
╚══════════════════════════════════════════╝
```

**ประโยชน์ 3 ข้อ:**
```
Internal Links
  ├── [1] เพิ่ม Time on Site
  │       ผู้อ่าน → อ่านต่อ → อ่านต่อ → อ่านต่อ
  ├── [2] Google เข้าใจโครงสร้าง
  │       บทความ A ←→ บทความ B ←→ บทความ C
  └── [3] กระจาย Link Juice
          หน้าแข็งแรง → ส่งพลังให้หน้าอื่น
```

**กฎ:**
- อย่างน้อย 3-5 Internal Links ต่อบทความ
- Anchor Text ต้องสื่อความหมาย (ไม่ใช่ "คลิกที่นี่")
- ลิงก์สองทาง: บทความใหม่ → เก่า + เก่า → ใหม่
- คลิกทดสอบทุกลิงก์ — ห้ามมี Broken Links (404)

## SLIDE 8: ตรวจ CTA Placement
```
╔══════════════════════════════════════════╗
║   ด้านที่ 5: CTA Placement (15 คะแนน)  ║
╚══════════════════════════════════════════╝
```

**3 ตำแหน่งแนะนำ:**
```
┌─────────────────────────────┐
│  Title + Intro              │
│  ┌───────────────────────┐  │
│  │  CTA #1 (Above Fold) │  │ ← จับคนที่สนใจทันที
│  └───────────────────────┘  │
│  Body Content...            │
│  Section ที่น่าสนใจที่สุด    │
│  ┌───────────────────────┐  │
│  │  CTA #2 (กลาง)       │  │ ← จับคนที่กำลัง Engage
│  └───────────────────────┘  │
│  Body Content ต่อ...        │
│  ┌───────────────────────┐  │
│  │  CTA #3 (ก่อน Conclusion)│ ← จับคนที่อ่านจนจบ
│  └───────────────────────┘  │
│  Conclusion                 │
└─────────────────────────────┘
```

**CTA ที่ดี = Value + Urgency + Clarity**

## SLIDE 9: Common Mistakes ครั้งที่ 2 (8 ข้อ)
```
╔══════════════════════════════════════════╗
║   Common Mistakes — 8 ข้อที่พบบ่อย     ║
╚══════════════════════════════════════════╝
```

| # | ข้อผิดพลาด | วิธีแก้ |
|---|-----------|--------|
| 1 | ไม่มี Meta Description | เขียนให้ครบ มี Keyword + CTA |
| 2 | Heading ผิดลำดับ (H1→H3) | ใช้ H1→H2→H3 ตามลำดับ |
| 3 | รูปไม่มี Alt Text + ใหญ่เกินไป | ใส่ Alt Text + Optimize ไม่เกิน 200 KB |
| 4 | ไม่ Preview บนมือถือ | ตรวจ 3 ขนาดหน้าจอทุกครั้ง |
| 5 | ไม่มี Internal Links | ใส่ 3-5 ลิงก์ที่เกี่ยวข้อง |
| 6 | CTA จุดเดียวตอนท้าย | วาง 2-3 จุด (ต้น กลาง ท้าย) |
| 7 | URL ยาวเกินไป | ทำให้สั้น มี Keyword |
| 8 | ไม่มี Conclusion | สรุปประเด็น + บอกทิศทาง |
