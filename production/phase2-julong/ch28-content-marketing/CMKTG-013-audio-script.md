**Host A:** สวัสดีครับทุกคน กลับมาพบกันอีกครั้งกับซีรีส์ Content Marketing Mastery ตอนที่ 13 แล้วครับ ตอนที่แล้วเราพูดถึง Template Blog Post List ซึ่งเป็นหน้ารวมบทความ วันนี้เราจะเข้าสู่หัวใจสำคัญของการเขียนบล็อกเลยครับ นั่นคือ **การสร้างบทความด้วย Content Builder** ใน Kartra ตอนที่ 1

**Host B:** ถูกต้องค่ะ ที่ผ่านมาเราพูดถึงโครงสร้าง Template กันเยอะแล้ว ทั้ง Blog Post Single, Blog Post List วันนี้ถึงเวลาลงมือเขียนจริงแล้วค่ะ Content Builder ของ Kartra เป็นเครื่องมือ Drag & Drop ที่ช่วยให้เราสร้างบทความได้โดยไม่ต้องเขียนโค้ดเลย แม้คนที่ไม่มีพื้นฐานเรื่องเว็บไซต์ก็ทำได้สบายๆ ค่ะ

**Host A:** วันนี้จะครอบคลุมตั้งแต่วิธีเข้าถึง Content Builder การเลือก Template เริ่มต้น การใช้ Drag & Drop Components ต่างๆ ไม่ว่าจะเป็น Text, Image, Video, Button, Divider จากนั้นเราจะพูดถึงการจัดรูปแบบ Typography การใส่ Internal Links และปิดท้ายด้วย SEO On-page ทั้ง Title, Meta Description และ URL Slug รวมถึงการ Preview กับ Publish ครับ

**Host B:** เริ่มจากขั้นตอนแรกเลยค่ะ คือ **วิธีเข้าถึง Content Builder** เมื่อเราล็อกอินเข้า Kartra Dashboard แล้ว เราไปที่เมนู "My Content" หรือ "Blog" จากนั้นกด "New Post" หรือ "Create New Article" ระบบจะพาเราเข้าสู่หน้าสร้างบทความใหม่ค่ะ ซึ่งจะมี 2 ส่วนหลักให้ตั้งค่า คือ ส่วน Settings กับส่วน Content Builder

**Host A:** ในส่วน Settings เราจะใส่ข้อมูลพื้นฐานก่อนครับ เช่น ชื่อบทความ หมวดหมู่ แท็ก และ Featured Image แต่เดี๋ยวเรื่องนี้เราจะกลับมาพูดตอนท้ายตอน SEO ส่วน Content Builder คือพื้นที่สร้างเนื้อหาจริงๆ ครับ เปรียบเหมือนกระดาษเปล่าที่เราจะเอาชิ้นส่วนต่างๆ มาวางเรียง

**Host B:** ก่อนเริ่มเขียน Kartra จะให้เราเลือก **Template** ก่อนค่ะ มี Template สำเร็จรูปมาให้หลายแบบ ทั้งแบบ Clean Minimal, แบบมี Sidebar, แบบ Full Width, แบบมี Hero Section ข้อดีคือเราไม่ต้องเริ่มจากศูนย์ค่ะ เลือก Template ที่ใกล้เคียงกับที่ต้องการ แล้วค่อยปรับแต่งทีหลัง

**Host A:** คำแนะนำสำหรับมือใหม่ครับ เลือก Template ที่ **เรียบง่าย** ก่อน อย่าเลือกแบบซับซ้อนมากเกินไป เพราะถ้ามี Component เยอะเราจะสับสนว่าอะไรแก้ตรงไหน เริ่มจาก Template เรียบๆ แล้วค่อยเพิ่ม Component ที่ต้องการทีละตัว จะเข้าใจหลักการได้เร็วกว่าครับ

**Host B:** พอเลือก Template แล้วก็เข้าสู่ **Content Builder** เต็มตัวค่ะ หน้าตาจะคล้ายๆ กับ Page Builder ที่เราเคยใช้สร้าง Landing Page ใน Kartra เลย ด้านซ้ายจะมี **Component Panel** ที่แสดง Components ทั้งหมดที่ใช้ได้ ส่วนตรงกลางคือ **Canvas** พื้นที่สร้างเนื้อหาจริงค่ะ

**Host A:** มาพูดถึง **Components** หลักๆ ที่เราจะใช้บ่อยที่สุดกันครับ ตัวแรกเลยคือ **Text Component** นี่คือหัวใจของบทความเลย เราใช้สำหรับเขียนเนื้อหาทั้งหมด ไม่ว่าจะเป็นหัวข้อ H1 H2 H3 ย่อหน้า รายการ Bullet Point หรือ Numbered List ทั้งหมดอยู่ใน Text Component ตัวเดียว วิธีใช้ง่ายมากครับ แค่ลาก Text Component จากแผงด้านซ้ายมาวางบน Canvas แล้วก็พิมพ์ได้เลย

**Host B:** ตัวที่สองคือ **Image Component** ค่ะ บทความที่ดีต้องมีรูปภาพประกอบ ไม่งั้นผู้อ่านจะเบื่อ ใน Kartra เราลาก Image Component มาวาง จากนั้นอัปโหลดรูป หรือใส่ URL ของรูปจากภายนอกก็ได้ค่ะ สิ่งสำคัญคืออย่าลืมใส่ **Alt Text** ทุกครั้ง เพราะมันช่วย SEO และช่วยผู้ที่ใช้ Screen Reader ด้วยค่ะ

**Host A:** นอกจากรูปภาพแล้ว **Video Component** ก็สำคัญไม่แพ้กันครับ โดยเฉพาะถ้าเราทำคอนเทนต์แบบ Multimedia เราสามารถ Embed วิดีโอจาก YouTube, Vimeo หรืออัปโหลดวิดีโอตรงเข้า Kartra ก็ได้ วิดีโอช่วยเพิ่ม Time on Page ได้มาก ซึ่ง Google ชอบมาก เพราะมันบอกว่าผู้อ่านสนใจเนื้อหาจริงๆ ครับ

**Host B:** คำถามที่หลายคนถามค่ะ ควร Embed จาก YouTube หรืออัปโหลดตรง คำตอบคือถ้าวิดีโออยู่บน YouTube อยู่แล้วก็ Embed เลยค่ะ ได้ประโยชน์ทั้ง YouTube View และไม่เปลืองพื้นที่ Storage แต่ถ้าเป็นวิดีโอที่ไม่อยากให้อยู่บน YouTube เช่น วิดีโอเฉพาะสมาชิก ก็อัปโหลดตรงเข้า Kartra จะดีกว่าค่ะ

**Host A:** ต่อมาคือ **Button Component** ครับ ปุ่มนี้สำคัญมากเพราะเป็นตัวนำพาผู้อ่านไปทำสิ่งที่เราต้องการ ไม่ว่าจะเป็นกดสมัครอีเมล กดไปยังหน้าสินค้า กดดาวน์โหลดไฟล์ หรือกดไปยังบทความอื่น เราสามารถตั้งค่าสี ขนาด ข้อความบนปุ่ม และ Link ปลายทางได้ทั้งหมดครับ

**Host B:** เคล็ดลับเรื่องปุ่มค่ะ ข้อความบนปุ่มต้อง **Action-Oriented** คือบอกว่ากดแล้วจะได้อะไร เช่น "ดาวน์โหลดฟรี" "สมัครเลย" "อ่านเพิ่มเติม" ดีกว่าแค่เขียนว่า "คลิกที่นี่" ค่ะ และสีของปุ่มควรตัดกับสีพื้นหลังให้ชัดเจน เพื่อให้สังเกตเห็นได้ง่าย

**Host A:** และ Component สุดท้ายที่ขาดไม่ได้คือ **Divider Component** ครับ ฟังดูเล็กน้อย แต่ Divider หรือเส้นคั่นช่วยแบ่งส่วนเนื้อหาให้ชัดเจน ทำให้บทความอ่านง่ายขึ้นมาก โดยเฉพาะบทความยาวๆ ถ้าไม่มี Divider เนื้อหาจะเป็นก้อนใหญ่ก้อนเดียว ผู้อ่านมองไม่ออกว่าตอนไหนเป็นเรื่องอะไร

**Host B:** นอกจาก 5 Components หลักแล้ว ยังมี Components อื่นๆ อีกค่ะ เช่น Columns สำหรับแบ่ง Layout เป็นหลายคอลัมน์, Spacer สำหรับเว้นระยะห่าง, Code Embed สำหรับใส่ HTML หรือ JavaScript เพิ่มเติม แต่สำหรับตอนที่ 1 นี้เราเน้น 5 ตัวหลักก่อนค่ะ ค่อยๆ เรียนรู้ทีละตัว

**Host A:** ทีนี้มาพูดถึง **Typography** หรือการจัดรูปแบบตัวอักษรครับ นี่คือสิ่งที่ทำให้บทความดูเป็นมืออาชีพ Typography ที่ดีประกอบด้วย **Heading Hierarchy** คือลำดับหัวข้อจาก H1 H2 H3 H4 โดย H1 ใช้สำหรับชื่อบทความมีแค่ 1 ตัว H2 สำหรับหัวข้อหลัก H3 สำหรับหัวข้อย่อย การจัดลำดับถูกต้องช่วย SEO ด้วยครับ

**Host B:** นอกจาก Heading แล้ว ยังมีเรื่อง **Font Size**, **Line Height**, **Font Weight** และ **Color** ค่ะ Body Text แนะนำขนาด 16-18px เพื่อให้อ่านง่ายบนทุกอุปกรณ์ Line Height แนะนำ 1.5-1.8 เท่า อย่าแน่นเกินไปจะอ่านลำบาก และสำคัญที่สุดคือ **ความสม่ำเสมอ** ค่ะ ใช้ Font Style เดียวกันตลอดทั้งบทความ อย่าเปลี่ยนไปมา

**Host A:** มาถึงเรื่อง **Internal Links** ครับ นี่คือเทคนิคที่หลายคนมองข้าม Internal Links คือการใส่ลิงก์ในบทความที่เชื่อมไปยังบทความอื่นๆ ในบล็อกเดียวกัน ข้อดีมี 3 อย่างครับ หนึ่ง ช่วยให้ผู้อ่านค้นพบเนื้อหาที่เกี่ยวข้อง สอง เพิ่ม Pages per Session ซึ่ง Google ชอบ และสาม ช่วยกระจาย Link Juice ให้ทุกหน้าแข็งแรงขึ้นในแง่ SEO

**Host B:** วิธีใส่ Internal Links ใน Content Builder ง่ายมากค่ะ Select ข้อความที่ต้องการ แล้วกดปุ่ม Link จากนั้นเลือก URL ของบทความปลายทาง เคล็ดลับคือ Anchor Text หรือข้อความที่เป็นลิงก์ ควรเป็นคำที่สื่อความหมายว่าคลิกไปแล้วจะเจออะไร เช่น "อ่านเพิ่มเติมเรื่อง Email Marketing" ดีกว่า "คลิกที่นี่" ค่ะ และแนะนำใส่ 3-5 Internal Links ต่อบทความค่ะ

**Host A:** ทีนี้มาถึงส่วนที่สำคัญมากครับ คือ **SEO On-page** สำหรับบทความแต่ละชิ้น มี 3 จุดหลักที่ต้องตั้งค่าคือ Title, Meta Description และ URL Slug เริ่มจาก **Title** ก่อนครับ Title คือชื่อบทความที่จะแสดงบน Google ควรมี Keyword หลักอยู่ในชื่อ ความยาวไม่เกิน 60 ตัวอักษร และต้องดึงดูดให้คนอยากคลิก

**Host B:** ต่อมาคือ **Meta Description** ค่ะ คือข้อความสั้นๆ ที่อยู่ใต้ชื่อบทความบนหน้า Google ความยาวแนะนำ 150-160 ตัวอักษร ต้องมี Keyword หลัก อธิบายสั้นๆ ว่าบทความเกี่ยวกับอะไร และมี Call to Action เบาๆ เพื่อให้คนอยากคลิกเข้ามาอ่าน เช่น "เรียนรู้วิธีสร้างบทความใน 10 นาที พร้อมเทคนิค SEO ที่ใช้ได้จริง" แบบนี้ค่ะ

**Host A:** และสุดท้ายคือ **URL Slug** ครับ URL Slug คือส่วนท้ายของ URL ที่บอกว่าหน้านี้เกี่ยวกับอะไร เช่น yoursite.com/blog/content-builder-guide ต้องสั้น ใช้ภาษาอังกฤษ ใส่ Keyword หลัก คั่นด้วยเครื่องหมายขีด ไม่มีอักขระพิเศษ และไม่ควรเปลี่ยนหลังจาก Publish แล้วครับ เพราะจะทำให้เกิด Broken Links

**Host B:** หลังจากสร้างเนื้อหาเสร็จและตั้งค่า SEO ครบแล้ว ก่อน Publish ต้อง **Preview** ก่อนเสมอค่ะ Kartra มีโหมด Preview ให้ดูว่าบทความจะแสดงผลเป็นอย่างไรบนหน้าจริง ตรวจสอบทั้ง Desktop, Tablet และ Mobile ดูว่ารูปภาพแสดงถูกต้อง ลิงก์ทำงาน ปุ่มกดได้ Typography อ่านง่าย และที่สำคัญคือ อ่านทั้งบทความจากต้นจนจบอย่างน้อยหนึ่งรอบค่ะ

**Host A:** และเมื่อ Preview เรียบร้อยแล้วก็ถึงขั้นตอนสุดท้ายครับ คือการ **Publish** กดปุ่ม Publish แล้วบทความจะไปอยู่บนหน้า Blog ของเราทันที แต่ก่อนกด Publish ให้ตรวจสอบ Checklist อีกรอบนะครับ ทั้ง Title, Meta Description, URL Slug, Featured Image, Category และ Tags ครบถ้วนแล้ว ก็กด Publish ได้เลย

**Host B:** สรุปแล้ว Content Builder ใน Kartra ประกอบด้วยขั้นตอนหลักๆ ค่ะ คือ เข้าถึง Builder ผ่าน New Post → เลือก Template → ใช้ Drag & Drop วาง Components 5 ตัวหลัก (Text, Image, Video, Button, Divider) → จัดรูปแบบ Typography → ใส่ Internal Links → ตั้งค่า SEO On-page (Title, Meta Description, URL Slug) → Preview ทุกขนาดจอ → Publish ค่ะ

**Host A:** ทุกอย่างเชื่อมกันเป็นระบบครับ เนื้อหาดี Typography ดี SEO ดี ก็เท่ากับบทความที่มีคุณภาพทั้งสำหรับผู้อ่านและสำหรับ Google ตอนหน้าเราจะมา ตรวจการบ้าน กันครับ เอาบทความที่สร้างมาดูว่าทำได้ตามที่เรียนไหม

**Host B:** ใช่ค่ะ ครั้งหน้าจะเป็นตอนตรวจการบ้านครั้งที่ 2 ค่ะ เตรียมบทความมาให้พร้อมนะคะ แล้วพบกันใหม่ค่ะ

**Host A:** บ๊ายบายครับ
