## คำถาม

### Q1. Custom CSS ใน Content Builder ใช้ทำอะไรได้ และข้อควรระวังคืออะไร?
- A) ใช้เปลี่ยนเนื้อหาบทความ ไม่ต้องระวังอะไร
- B) ใช้ปรับแต่ง สี Font Border Shadow ให้ตรง Brand + ต้อง Test Responsive บนมือถือทุกครั้ง
- C) ใช้ได้เฉพาะเปลี่ยนสีตัวอักษรเท่านั้น
- D) ใช้ได้เฉพาะบน Desktop ไม่ต้องสนใจมือถือ

### Q2. การ Embed วิดีโอ YouTube ในบทความ ข้อใดเป็นแนวปฏิบัติที่ถูกต้อง?
- A) Embed ได้ไม่จำกัดจำนวน ยิ่งเยอะยิ่งดี
- B) ใช้ Responsive Embed + Lazy Loading และไม่เกิน 2-3 Embed ต่อบทความ
- C) ไม่ต้องสนใจ Responsive เพราะ YouTube จัดการให้อัตโนมัติ
- D) ควร Embed เฉพาะ Vimeo เท่านั้น YouTube ไม่รองรับ

### Q3. Content Upgrade ต่างจาก Lead Magnet ทั่วไปอย่างไร?
- A) Content Upgrade ราคาแพงกว่า Lead Magnet
- B) Content Upgrade เจาะจงกับแต่ละบทความ มี Conversion สูงกว่า Lead Magnet ทั่วไป 5-10 เท่า
- C) Content Upgrade ไม่ต้องแลกอีเมล ดาวน์โหลดได้เลย
- D) ไม่มีความแตกต่าง เป็นคำเดียวกัน

### Q4. ตำแหน่งที่ดีที่สุดสำหรับวาง Content Upgrade (Opt-in Form) ในบทความคือที่ไหน?
- A) ต้นบทความเท่านั้น
- B) ท้ายบทความเท่านั้น
- C) หลังจากเนื้อหาส่วนที่น่าสนใจที่สุดในบทความ
- D) ใน Sidebar ของเว็บไซต์

### Q5. Social Share Buttons ควรมีกี่แพลตฟอร์ม และต้องตั้งอะไรเพิ่มเติม?
- A) 2 แพลตฟอร์ม (FB, LINE) ไม่ต้องตั้งอะไรเพิ่ม
- B) 5 แพลตฟอร์ม (FB/LINE/X/LinkedIn/Copy Link) + ต้องตั้ง Open Graph Tags ให้ถูกต้อง
- C) 10 แพลตฟอร์มทุกอันที่มี ไม่ต้องสนใจ OG Tags
- D) ไม่จำเป็นต้องมี Social Share Buttons เลย

### Q6. Blog Analytics ผ่าน GA4 ต้องดูตัวเลขอะไรบ้าง?
- A) แค่ Page Views อย่างเดียวก็พอ
- B) Page Views, Average Engagement Time, Bounce Rate, Scroll Depth, Conversion Rate
- C) แค่จำนวน Comments กับ Shares
- D) Revenue เท่านั้น

### Q7. Image Compression ที่ถูกต้องควรทำอย่างไร?
- A) ใช้ PNG ความละเอียดสูงสุดเท่าที่ได้ ไม่ต้อง Compress
- B) ใช้ WebP (เล็กกว่า 30-50%) + Compress ด้วย TinyPNG/Squoosh + ไม่เกิน 1200px กว้าง
- C) ลดขนาดรูปจนมองไม่เห็นเพื่อให้โหลดเร็ว
- D) ไม่ต้องใส่รูปในบทความเลย จะได้โหลดเร็วที่สุด

### Q8. Lazy Loading คืออะไร และช่วยเพิ่ม Page Speed ได้เท่าไร?
- A) เทคนิคที่โหลดทุกอย่างล่วงหน้าเพื่อให้เร็วขึ้น — เพิ่ม 5 คะแนน
- B) เทคนิคที่โหลดรูป/Embed เมื่อ Scroll มาถึงเท่านั้น — เพิ่ม 20-40 คะแนน
- C) เทคนิคที่ลดคุณภาพรูปให้โหลดเร็วขึ้น — เพิ่ม 10 คะแนน
- D) เทคนิคที่ซ่อนรูปภาพทั้งหมด — ไม่เกี่ยวกับ Page Speed

### Q9. หน้าเว็บที่โหลดช้าเกิน 3 วินาที จะสูญเสียผู้อ่านกี่เปอร์เซ็นต์?
- A) 10%
- B) 25%
- C) 53%
- D) 80%

### Q10. Content Calendar ควร Publish บทความบ่อยแค่ไหน?
- A) 10 บทความต่อวันเพื่อให้ Google ชอบ
- B) 1-2 บทความต่อสัปดาห์อย่างสม่ำเสมอ
- C) 1 บทความต่อเดือนก็เพียงพอ
- D) ไม่ต้องมี Calendar Publish เมื่อไรก็ได้

### Q11. Repurpose Blog เป็น Social Media Content ทำได้กี่รูปแบบหลักๆ?
- A) 3 แบบ (รูป ข้อความ วิดีโอ)
- B) 7 แบบ (Carousel, Quote Card, Thread, Podcast, Short Video, Infographic, Newsletter)
- C) 1 แบบ คือแชร์ลิงก์บทความไปใน Social Media
- D) ไม่สามารถ Repurpose ได้ ต้องสร้างใหม่ทุกครั้ง

### Q12. หลักการสำคัญที่สุดของ Content Repurposing คืออะไร?
- A) สร้างคอนเทนต์ใหม่ทุกครั้งสำหรับทุก Platform
- B) สร้างครั้งเดียว แจกจ่ายหลายที่ — 1 บทความ = 10-15 ชิ้นคอนเทนต์
- C) Copy เนื้อหาเดิมไปวางทุก Platform โดยไม่ปรับ
- D) Repurpose เฉพาะบทความที่ได้ยอด Views สูง

## เฉลย

| ข้อ | คำตอบ | อธิบาย |
|-----|-------|--------|
| Q1 | ✅ B) ปรับแต่ง สี Font Border Shadow + Test Responsive | Custom CSS ทำให้บทความโดดเด่นตรง Brand Identity แต่ต้อง Test บนมือถือทุกครั้งเพราะ CSS อาจแสดงผลต่างกันในแต่ละหน้าจอ |
| Q2 | ✅ B) Responsive Embed + Lazy Loading ไม่เกิน 2-3 | Embed มากเกินไปทำให้หน้าโหลดช้า ต้อง wrap ด้วย Container ที่มี padding-bottom: 56.25% สำหรับ 16:9 และใช้ Lazy Loading |
| Q3 | ✅ B) เจาะจงแต่ละบทความ Conversion สูงกว่า 5-10 เท่า | Content Upgrade เป็น Lead Magnet ที่เกี่ยวข้องโดยตรงกับบทความที่กำลังอ่าน ทำให้ผู้อ่านรู้สึกว่าได้ค่ามากกว่า |
| Q4 | ✅ C) หลังจากเนื้อหาส่วนที่น่าสนใจที่สุด | เพราะผู้อ่านกำลัง Engage สูงสุด ณ จุดนั้น ทำให้มีโอกาสกรอกอีเมลสูง ดีกว่าวางแค่ต้นหรือท้ายบทความ |
| Q5 | ✅ B) 5 แพลตฟอร์ม + Open Graph Tags | FB/LINE/X/LinkedIn/Copy Link ครอบคลุมผู้ใช้ไทย OG Tags ทำให้รูป/Title/Description แสดงสวยเมื่อแชร์ |
| Q6 | ✅ B) Page Views, Engagement Time, Bounce Rate, Scroll Depth, Conversion Rate | 5 ตัวเลขนี้ให้ภาพรวมครบว่าบทความดึงดูดคนได้ไหม คนอ่านจริงไหม และสร้าง Conversion ได้ไหม |
| Q7 | ✅ B) WebP + Compress + ไม่เกิน 1200px | WebP เล็กกว่า PNG/JPEG 30-50% คุณภาพเท่าเดิม ใช้ TinyPNG/Squoosh Compress + Srcset ให้ Browser เลือกขนาดเหมาะสม |
| Q8 | ✅ B) โหลดเมื่อ Scroll มาถึง เพิ่ม 20-40 คะแนน | Lazy Loading ไม่โหลดรูป/Embed ทั้งหมดตั้งแต่แรก ทำให้หน้าแรก Load เร็วขึ้นมาก วิธีทำง่ายแค่เพิ่ม loading="lazy" |
| Q9 | ✅ C) 53% | เกินครึ่งของผู้อ่านจะออกไปทันทีถ้าหน้าโหลดนานเกิน 3 วินาที Performance จึงสำคัญมาก |
| Q10 | ✅ B) 1-2 บทความ/สัปดาห์อย่างสม่ำเสมอ | Google ชอบเว็บที่อัปเดตสม่ำเสมอ ดีกว่า Publish เยอะแล้วหายไป ใช้ Batch Process + Schedule ช่วย |
| Q11 | ✅ B) 7 แบบ | Carousel, Quote Card, Thread, Podcast, Short Video, Infographic, Newsletter ทำให้ 1 บทความกลายเป็น 10-15 ชิ้นคอนเทนต์ |
| Q12 | ✅ B) สร้างครั้งเดียว แจกจ่ายหลายที่ | Blog คือ Pillar Content ที่ลงทุนสร้างอย่างดี แล้ว Repurpose เป็นรูปแบบต่างๆ สำหรับแต่ละ Platform ไม่ใช่แค่ Copy ไปวาง แต่ปรับ Format ให้เหมาะสม |
