## SLIDE 3: ทบทวน Content Funnel
```
╔══════════════════════════════════════════╗
║      Content Funnel: TOFU/MOFU/BOFU     ║
╚══════════════════════════════════════════╝
```

```
         ╭──────────╮
         │   BOFU   │  10% — Conversion (ปิดการขาย)
         │  รีวิว   │  รีวิวจริง, Social Proof
        ╭┴──────────┴╮
        │    MOFU    │  30% — Consideration (สร้างความเชื่อมั่น)
        │  How-to    │  วิธีทำ, Case Study, เปรียบเทียบ
       ╭┴────────────┴╮
       │     TOFU     │  60% — Awareness (ดึงคนใหม่)
       │  กว้าง ง่าย  │  "...คืออะไร", "5 วิธี...", Search Volume สูง
       ╰──────────────╯
```

**ข้อผิดพลาดหลัก**:
- เขียน BOFU ก่อนโดยไม่มี TOFU = เปิดร้านแต่ไม่มีคนรู้จัก
- ปน TOFU + BOFU ในบทความเดียว = ไม่ได้ทั้งสองอย่าง

## SLIDE 4: ตรวจโครงสร้างบทความ (ปัญหาที่พบ)
```
╔══════════════════════════════════════════╗
║    ตรวจโครงสร้าง Blog: 5 ปัญหาหลัก     ║
╚══════════════════════════════════════════╝
```

| # | ปัญหา | สิ่งที่ควรเป็น |
|---|-------|--------------|
| 1 | ไม่มี H2/H3 เขียนเป็นก้อนยาว | H2 หัวข้อหลัก + H3 หัวข้อย่อย |
| 2 | ย่อหน้ายาว 10-15 บรรทัด | ไม่เกิน 3-4 บรรทัด |
| 3 | ไม่มี Table of Contents | ใส่สำหรับบทความ >1,500 คำ |
| 4 | ข้อมูลสำคัญอยู่ท้ายบทความ | Inverted Pyramid ข้อมูลสำคัญขึ้นก่อน |
| 5 | ไม่มี Visual / มีแค่รูปปก | Visual ทุก 300-400 คำ |

**สาเหตุ**: เขียนแบบ "เรียงความ" ไม่ใช่ "บทความออนไลน์"

## SLIDE 5: ตรวจ Headline — 4 จุดอ่อน
```
╔══════════════════════════════════════════╗
║    Headline: 4 ปัญหาที่พบบ่อย          ║
╚══════════════════════════════════════════╝
```

| จุดอ่อน | ตัวอย่างผิด | ตัวอย่างถูก |
|---------|-----------|-----------|
| ยาวเกิน 70 ตัวอักษร | Google ตัดข้อความออก | สั้นกระชับ แสดงครบในผลค้นหา |
| ไม่มี Keyword | "เปลี่ยนชีวิตด้วยพลังแห่งการเขียน" | "วิธีเขียน Blog หาเงินออนไลน์" |
| ไม่มี Power Words | "การทำ Content Marketing" | "เคล็ดลับ Content Marketing ที่คนส่วนใหญ่ไม่รู้" |
| ไม่สื่อ Benefit | "Content Marketing" | "Content Marketing สำหรับมือใหม่ — เริ่มฟรี" |

**Formula**: `[Power Word] + [Keyword] + [Benefit]` ไม่เกิน 70 ตัวอักษร

## SLIDE 6: ตรวจ Opening Hook — 4 เทคนิคที่ได้ผล
```
╔══════════════════════════════════════════╗
║    Opening Hook: 4 เทคนิค              ║
╚══════════════════════════════════════════╝
```

| เทคนิค | ตัวอย่าง |
|--------|---------|
| คำถาม | "เคยสงสัยไหมว่าทำไมบทความของคุณไม่มีคนอ่าน?" |
| สถิติ | "70% ของบทความออนไลน์ ไม่มีคนอ่านเกินย่อหน้าแรก" |
| เรื่องเล่า | "ผมเคยเขียน Blog 50 บทความ แต่ไม่มี Traffic สักคน" |
| ข้อเท็จจริงขัดสามัญสำนึก | "การเขียน Blog ไม่ต้องเก่งเขียน แต่ต้องเก่ง Research" |

**ห้ามทำ**: เปิดด้วยคำนิยาม เช่น "Content Marketing คือ..."

**ทดสอบ**: อ่าน Hook ออกเสียง ถ้าตื่นเต้น = ใช้ได้ ถ้าเฉยๆ = เขียนใหม่

## SLIDE 7: Bridge — เชื่อม Hook กับเนื้อหา
```
╔══════════════════════════════════════════╗
║    Bridge: เชื่อม Hook → เนื้อหาหลัก   ║
╚══════════════════════════════════════════╝
```

```
┌─────────────────────┐
│       HOOK          │ ← ดึงดูดผู้อ่าน (คำถาม/สถิติ/เรื่องเล่า)
│ "เคยสงสัยไหมว่า..." │
└─────────┬───────────┘
          │
    ┌─────┴─────┐
    │  BRIDGE   │ ← เชื่อม ทำให้เนื้อหาเป็น "คำตอบ"
    │ Transition│
    └─────┬─────┘
          │
┌─────────┴───────────┐
│    เนื้อหาหลัก      │ ← ตอบคำถามที่ Hook ตั้งไว้
│ ข้อมูล วิธีทำ ตัวอย่าง│
└─────────────────────┘
```

**ปัญหา**: Hook ดี แต่พอเข้าเนื้อหาเหมือนเปลี่ยนเรื่อง → ผู้อ่านสับสน ออกจากบทความ

## SLIDE 8: 7 Common Mistakes (ข้อ 1-4)
```
╔══════════════════════════════════════════╗
║    7 Common Mistakes (Part 1)           ║
╚══════════════════════════════════════════╝
```

| # | Mistake | ผลเสีย | วิธีแก้ |
|---|---------|--------|--------|
| 1 | เขียนเพื่อตัวเอง | เนื้อหาไม่ตรงกับผู้อ่าน | ถาม: ผู้อ่านคือใคร? มีปัญหาอะไร? |
| 2 | ไม่ทำ Keyword Research | ไม่มี Traffic | Research ก่อนเขียนทุกครั้ง |
| 3 | ไม่มี CTA / CTA ไม่เกี่ยว | เสียโอกาสธุรกิจ | CTA ต้อง Match เนื้อหา |
| 4 | ยาวไม่มีโครงสร้าง | Bounce Rate สูง | H2/H3 + ย่อหน้าสั้น + Visual |

## SLIDE 9: 7 Common Mistakes (ข้อ 5-7)
```
╔══════════════════════════════════════════╗
║    7 Common Mistakes (Part 2)           ║
╚══════════════════════════════════════════╝
```

| # | Mistake | ผลเสีย | วิธีแก้ |
|---|---------|--------|--------|
| 5 | ไม่ Proofread | ทำลายความน่าเชื่อถือ | อ่านทบทวน 2 รอบก่อน Publish |
| 6 | ก๊อปเนื้อหา / AI ไม่ปรับ | Google ลดอันดับ Duplicate | เขียนด้วยเสียงตัวเอง เพิ่มมุมมอง |
| 7 | ไม่มี Internal Links | ผู้อ่านออกทันที | ใส่ Internal Links 2-3 จุดต่อบทความ |

**Feedback อาจารย์**: นักเรียนเข้าใจทฤษฎีดี แต่ลงมือทำยังไม่ครบ
→ **แก้ด้วย Checklist** ตรวจทุกข้อก่อน Publish
