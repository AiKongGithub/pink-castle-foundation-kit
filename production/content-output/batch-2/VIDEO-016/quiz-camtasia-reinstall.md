<details>
<summary>View Answer</summary>

## Question 4

**Folder ใดไม่จำเป็นต้องลบใน Clean Uninstall?**

- A) %APPDATA%\TechSmith
- B) %LOCALAPPDATA%\TechSmith
- C) C:\ProgramData\TechSmith
- D) Documents\Camtasia

<details>
<summary>View Answer</summary>

**Correct Answer: D) Documents\Camtasia**

**Explanation:** Documents\Camtasia คือที่เก็บ Project files ซึ่งเป็นงานของผู้ใช้ ไม่ควรลบ

Folders ที่ต้องลบใน Clean Uninstall:
- %APPDATA%\TechSmith (User preferences)
- %LOCALAPPDATA%\TechSmith (Cache)
- C:\ProgramData\TechSmith (Shared data)

## Question 5

**Command อะไรใช้เปิด AppData\Roaming folder ใน Windows?**

- A) %TEMP%
- B) %APPDATA%
- C) %USERPROFILE%
- D) %PROGRAMFILES%

<details>
<summary>View Answer</summary>

**Correct Answer: B) %APPDATA%**

**Explanation:** Windows Commands:
- **%APPDATA%** = AppData\Roaming folder
- %LOCALAPPDATA% = AppData\Local folder
- %TEMP% = Temporary files
- %USERPROFILE% = User folder

วิธีใช้: กด Win+R แล้วพิมพ์ %APPDATA%

</details>

## Question 7

**ทำไมต้อง Download installer ใหม่ แม้จะมีไฟล์เก่าอยู่?**

- A) เพราะไฟล์เก่าหมดอายุ
- B) เพราะต้องจ่ายเงินซ้ำ
- C) เพราะได้ version ล่าสุดและไฟล์ไม่ corrupt
- D) เพราะ Windows บังคับ

<details>
<summary>View Answer</summary>

**Correct Answer: C) เพราะได้ version ล่าสุดและไฟล์ไม่ corrupt**

**Explanation:** เหตุผลที่ต้อง Download ใหม่:
1. ได้ version ล่าสุดที่มี bug fixes

ไม่เกี่ยวกับการจ่ายเงินหรือหมดอายุ

## Question 10

**ข้อใดเป็นลำดับที่ถูกต้องของ Clean Reinstall?**

- A) Uninstall → Download → Install → Backup
- B) Backup → Uninstall → Restart → Download → Install
- C) Download → Backup → Uninstall → Install
- D) Install → Uninstall → Backup → Restart

<details>
<summary>View Answer</summary>

**Correct Answer: B) Backup → Uninstall → Restart → Download → Install**

**Explanation:** ลำดับที่ถูกต้อง:
1. **Backup** - Project files, License Key
2. **Uninstall** - Standard + Clean leftover files
3. **Restart** - Clear system state
4. **Download** - Latest version
5. **Install** - Fresh installation

Backup ต้องทำก่อนเสมอ!

</details>
