## Key Concepts

### 1. Website Design Principles
- **Visual Hierarchy**: จัดลำดับความสำคัญด้วยสายตา
- **White Space**: พื้นที่ว่างช่วยให้อ่านง่าย
- **Consistency**: ใช้รูปแบบเดียวกันทั้งเว็บ
- **Mobile-First**: ออกแบบสำหรับมือถือก่อน

### 2. Color Psychology
- **Red/Orange**: กระตุ้นการตัดสินใจ (CTA)
- **Blue**: ความน่าเชื่อถือ (ธนาคาร, Tech)
- **Green**: สุขภาพ, ความสำเร็จ
- **Purple**: หรูหรา, Premium
- **Yellow**: ดึงดูดความสนใจ

### 3. Typography Basics
- **Heading**: Sans-serif ตัวหนา
- **Body**: 16px minimum สำหรับ mobile
- **Line Height**: 1.5-1.7 อ่านสบาย
- **Contrast**: ตัวหนังสือต้องชัด

### 4. Landing Page Anatomy
1. **Hero Section**: Headline + Subheadline + CTA
2. **Problem**: ปัญหาที่ลูกค้าเจอ
3. **Solution**: วิธีแก้ปัญหา
4. **Benefits**: ประโยชน์ที่ได้รับ
5. **Social Proof**: Testimonials, Reviews
6. **FAQ**: คำถามที่พบบ่อย
7. **CTA**: Call-to-Action สุดท้าย

### 5. Sales Page Elements
- **Headline Formula**: Result + Time + Objection
- **Story**: เล่าเรื่องที่ connect กับลูกค้า
- **Features vs Benefits**: เน้น Benefits
- **Price Anchoring**: แสดงมูลค่าเทียบราคา
- **Guarantee**: รับประกันความพอใจ
- **Urgency/Scarcity**: จำกัดเวลา/จำนวน

### 6. Conversion Rate Optimization (CRO)
- **A/B Testing**: ทดสอบ 2 versions
- **Heatmaps**: ดูว่าคนคลิกตรงไหน
- **Form Optimization**: ลดจำนวน fields
- **Page Speed**: ทุก 1 วินาที = 7% conversion
- **Mobile Optimization**: 60%+ traffic จากมือถือ

### 7. Trust Elements
- **SSL Certificate**: https:// และรูปกุญแจ
- **Testimonials**: รีวิวจากลูกค้าจริง
- **Case Studies**: ผลลัพธ์ที่พิสูจน์ได้
- **Media Logos**: "As seen on..."
- **Certifications**: ใบรับรอง, รางวัล
- **Contact Info**: ที่อยู่, เบอร์โทร (มีตัวตน)

### 8. CTA Best Practices
- **Above the Fold**: เห็น CTA ทันทีไม่ต้อง scroll
- **Contrasting Color**: สีตัดกับ background
- **Action Words**: "Get", "Start", "Download"
- **Benefit-Focused**: "Get Free Guide" ไม่ใช่ "Submit"
- **Multiple CTAs**: วางหลายจุดในหน้ายาว

### 9. Google Analytics Setup
- **GA4**: Google Analytics 4 (ใหม่)
- **Key Metrics**: Sessions, Users, Bounce Rate
- **Goals**: ตั้งเป้าหมาย Conversion
- **Events**: Track การคลิก, Form submit
- **UTM Parameters**: Track แหล่งที่มา

### 10. Website Maintenance
- **Weekly**: Check analytics, Reply comments
- **Monthly**: Update plugins, Security scan
- **Quarterly**: Content audit, Speed test
- **Yearly**: Renew domain/hosting, Backup review

## Frameworks

### AIDA for Landing Pages
```
Attention → Interest → Desire → Action
```

### PAS for Sales Copy
```
Problem → Agitate → Solution
```

### 80/20 Rule
```
20% ของหน้าสร้าง 80% ของ conversions
เน้น optimize หน้าสำคัญก่อน
```
