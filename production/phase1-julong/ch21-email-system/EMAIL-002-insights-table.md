## ตาราง 2: องค์ประกอบ Email Header (Content Structure)

| องค์ประกอบ | ตำแหน่ง | หน้าที่ | แนวปฏิบัติที่ดี |
|-----------|---------|---------|---------------|
| โลโก้ | ตรงกลางหรือมุมซ้ายบน | สร้าง Brand Recognition | ขนาดไม่ใหญ่เกินไป คมชัด |
| สีแบรนด์ | พื้นหลังหรือเส้นคั่น | สร้างความคุ้นเคย | ใช้สีเดียวกับเว็บไซต์ |
| ลิงก์นำทาง | ใต้โลโก้หรือด้านข้าง | นำทางไปเว็บ/บล็อก | ไม่เกิน 3-4 ลิงก์ |
| Pre-header Text | ไม่แสดงใน Header แต่แสดงใน inbox | เสริม Subject Line | 40-100 ตัวอักษร |

## ตาราง 5: HTML vs Plain Text Email (Comparison Table)

| เกณฑ์ | HTML Email | Plain Text Email |
|-------|-----------|-----------------|
| รูปแบบ | มีสี รูปภาพ ปุ่ม CTA | ข้อความล้วน ไม่มีการจัดรูปแบบ |
| ข้อดี | ดูเป็นมืออาชีพ สวยงาม | ดูจริงใจ เหมือนเขียนถึงเอง |
| ข้อเสีย | อาจแสดงผลต่างกันในแต่ละ client | ดูไม่น่าสนใจ ไม่มีภาพ |
| Deliverability | อาจเข้า Promotion tab | มักเข้า Primary inbox |
| เหมาะกับ | Newsletter, โปรโมชัน, ประกาศ | Follow-up ส่วนตัว, ขาย B2B |
| สร้างยาก/ง่าย | ต้องมีทักษะ HTML/Design | เขียนได้ทันที |
| Mobile | ต้องออกแบบ Responsive | แสดงผลได้ทุกอุปกรณ์ |

## ตาราง 6: Mobile Responsive Checklist (Process Table)

| รายการตรวจสอบ | เกณฑ์ที่ดี | เกณฑ์ที่ไม่ดี |
|-------------|----------|-------------|
| ความกว้าง Header | ปรับตามขนาดหน้าจอ | กว้างคงที่ 600px+ |
| ขนาดตัวอักษร | 14px ขึ้นไป | เล็กกว่า 12px |
| ปุ่ม CTA | ใหญ่พอกดด้วยนิ้ว (44x44px+) | เล็กเกินไป กดยาก |
| รูปภาพ | ปรับขนาดอัตโนมัติ | ล้นออกนอกจอ |
| คอลัมน์ | เรียงเป็นแถวเดียวบนมือถือ | หลายคอลัมน์บีบอัดกัน |
| ช่องว่าง | มีที่ว่างให้หายใจ | แน่นเกินไป อ่านยาก |
