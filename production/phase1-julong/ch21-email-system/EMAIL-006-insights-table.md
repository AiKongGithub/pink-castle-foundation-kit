## ตาราง 3: DNS Records สำหรับอีเมล (Reference Table)

| Record | ชื่อเต็ม | หน้าที่ | ตัวอย่างค่า | ความสำคัญ |
|--------|---------|--------|-----------|----------|
| SPF | Sender Policy Framework | ระบุเซิร์ฟเวอร์ที่มีสิทธิ์ส่งอีเมล | `v=spf1 +a +mx ~all` | สูงมาก |
| DKIM | DomainKeys Identified Mail | ลายเซ็นดิจิทัลยืนยันตัวตน | public key ในรูป TXT record | สูงมาก |
| DMARC | Domain-based Message Authentication | นโยบายจัดการอีเมลที่ไม่ผ่าน | `v=DMARC1; p=none; rua=mailto:...` | สูง |
| MX | Mail Exchange | กำหนดเส้นทางอีเมลเข้า | `mail.yourdomain.com` | สูงมาก |
| PTR | Pointer Record | Reverse DNS สำหรับ IP | ตั้งโดยผู้ให้บริการโฮสติ้ง | ปานกลาง |

## ตาราง 4: เปรียบเทียบ SPF, DKIM, DMARC (Comparison Table)

| เกณฑ์ | SPF | DKIM | DMARC |
|-------|-----|------|-------|
| เปรียบเทียบ | ยาม (Guard) | ตราประทับ (Seal) | กฎหมาย (Law) |
| ป้องกันอะไร | การปลอมแปลงผู้ส่ง | การแก้ไขเนื้อหา | การใช้โดเมนโดยไม่ได้รับอนุญาต |
| ตั้งค่าที่ไหน | DNS TXT Record | DNS TXT Record | DNS TXT Record |
| ความยาก | ง่าย | ปานกลาง | ง่าย-ปานกลาง |
| จำเป็นไหม | จำเป็นมาก | จำเป็นมาก | แนะนำอย่างยิ่ง |
| ถ้าไม่มี | อีเมลเข้า Spam | ไม่ได้ยืนยันตัวตน | ไม่มีนโยบายป้องกัน |

## ตาราง 5: ปัญหาที่พบบ่อยและวิธีแก้ (Troubleshooting Table)

| ปัญหา | อาการ | สาเหตุ | วิธีแก้ไข |
|-------|-------|--------|---------|
| อีเมลเข้า Spam | ผู้รับเห็นอีเมลในกล่อง Spam | ไม่มี SPF/DKIM/DMARC | ตั้งค่า DNS Records ทั้ง 3 ตัว |
| อีเมลส่งไม่ออก | กด Send แล้วเจอ Error | รหัสผ่านผิด / SMTP ปิด | ตรวจรหัสผ่าน ติดต่อโฮสติ้ง |
| อีเมล Bounce กลับ | ได้รับ Delivery Failure | ที่อยู่ผิด / กล่องเต็ม | ตรวจสอบที่อยู่ แจ้งผู้รับ |
| ไฟล์แนบไม่ถึง | ผู้รับไม่เห็นไฟล์แนบ | ไฟล์ใหญ่เกิน / นามสกุลถูกบล็อก | ลดขนาดไฟล์ เปลี่ยนนามสกุล |
| แสดงผลผิดเพี้ยน | อีเมลอ่านยากบนมือถือ | ใช้ HTML ซับซ้อนเกินไป | ใช้ข้อความ plain text หรือ HTML เรียบง่าย |
| ลิงก์ไม่ทำงาน | คลิกลิงก์แล้วไม่เปิดหน้า | URL ผิด / ถูกบล็อก | ตรวจสอบ URL ให้ถูกต้อง |
