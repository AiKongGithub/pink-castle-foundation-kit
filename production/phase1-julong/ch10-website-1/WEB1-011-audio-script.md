## เปิดรายการ

**Host A:** สวัสดีครับทุกคน กลับมาพบกันอีกครั้งกับรายการ PinkCastle Academy Podcast ตอนที่แล้วเราพูดถึง WP-Rocket ที่ช่วยเรื่องความเร็วเว็บไซต์ วันนี้เรามาต่อกันเรื่อง "การปรับแต่ง WordPress Admin Panel" ครับ ซึ่งเป็นเรื่องที่หลายคนมองข้ามแต่สำคัญมากครับ

**Host B:** ใช่เลยค่ะ WordPress Admin Panel หรือหน้าหลังบ้านนี่แหละคือศูนย์บัญชาการของเว็บไซต์เราค่ะ ถ้าเราจัดระเบียบและตั้งค่ามันให้ดี การบริหารเว็บก็จะง่ายขึ้นมากเลยค่ะ

## ช่วงที่ 1: Dashboard Widgets

**Host A:** มาเริ่มจาก Dashboard ก่อนเลยครับ หน้า Dashboard คือหน้าแรกที่เห็นเมื่อเข้า Admin Panel จะมี Widget หลายตัวแสดงอยู่ เช่น At a Glance ที่บอกจำนวนโพสต์ หน้า และความคิดเห็น Quick Draft สำหรับเขียนร่างโพสต์เร็วๆ Activity ที่แสดงกิจกรรมล่าสุด และ WordPress Events and News ครับ

**Host B:** หลายคนไม่รู้ว่า Widget เหล่านี้จัดการได้ค่ะ กดที่ Screen Options ตรงมุมขวาบน แล้วติ๊กเลือกว่าจะแสดง Widget ไหนบ้าง Widget ที่ไม่ได้ใช้ก็ปิดไปเลย เพื่อให้หน้า Dashboard สะอาดและโฟกัสกับข้อมูลที่สำคัญจริงๆ ค่ะ

**Host A:** นอกจากนี้ยังลาก Widget เพื่อจัดเรียงลำดับได้ด้วยนะครับ อยากให้ Widget ไหนอยู่ด้านบนสุดก็ลากขึ้นไปได้เลย ทำให้หน้า Dashboard ของเราเป็นระเบียบและใช้งานง่ายขึ้นครับ

---

## ช่วงที่ 2: User Roles & Permissions

**Host B:** มาต่อเรื่องที่สำคัญมากค่ะ คือ User Roles หรือบทบาทของผู้ใช้ WordPress มี 5 บทบาทหลัก ได้แก่ Administrator ที่มีสิทธิ์ทำทุกอย่าง Editor ที่จัดการเนื้อหาได้ทั้งหมดรวมถึงโพสต์ของคนอื่น Author ที่เขียนและเผยแพร่โพสต์ของตัวเองได้ Contributor ที่เขียนได้แต่ไม่สามารถเผยแพร่เอง และ Subscriber ที่อ่านได้อย่างเดียวค่ะ

**Host A:** เรื่องนี้สำคัญมากครับ โดยเฉพาะถ้าเว็บไซต์มีทีมงานหลายคน อย่าให้ทุกคนเป็น Administrator เด็ดขาดครับ เพราะ Administrator มีสิทธิ์ลบ Plugin ลบ Theme หรือแม้แต่ลบเว็บทั้งหมดได้ ให้สิทธิ์เท่าที่จำเป็นเท่านั้นครับ

**Host B:** ยกตัวอย่างเช่น ถ้าจ้างนักเขียนมาช่วยเขียนบทความ ให้ตั้งเป็น Author ก็พอค่ะ ไม่ต้องเป็น Editor ถ้าเป็นคนที่ช่วยส่งบทความมาให้ตรวจก่อนเผยแพร่ ตั้งเป็น Contributor ก็เพียงพอค่ะ

---

## ช่วงที่ 3: Admin Menu & Writing Settings

**Host A:** ต่อมาเรื่อง Admin Menu ครับ เราสามารถปรับแต่งเมนูด้านซ้ายของ Admin Panel ได้ด้วย Plugin เช่น Admin Menu Editor สามารถซ่อนเมนูที่ไม่ได้ใช้ เปลี่ยนชื่อเมนู หรือจัดลำดับเมนูใหม่ได้ครับ ช่วยให้ทำงานเร็วขึ้น

**Host B:** ส่วนเรื่อง Writing Settings ก็สำคัญไม่แพ้กันค่ะ เข้าไปที่ Settings แล้วเลือก Writing ตรงนี้เราสามารถตั้งค่า Default Post Category ว่าโพสต์ใหม่จะอยู่หมวดหมู่อะไรเป็นค่าเริ่มต้น และ Default Post Format ค่ะ

**Host A:** อีกเรื่องที่สำคัญมากคือ Permalinks ครับ เข้าไปที่ Settings แล้วเลือก Permalinks แนะนำให้ตั้งเป็น Post name เช่น yoursite.com/sample-post เพราะอ่านง่ายและดีต่อ SEO ครับ อย่าใช้แบบ Default ที่เป็นตัวเลข เช่น ?p=123 เพราะ SEO ไม่ดีเลยครับ

---

## ช่วงที่ 4: Reading & Discussion Settings

**Host B:** มาดู Reading Settings กันค่ะ ตรงนี้สำคัญมากเลย เพราะเราเลือกได้ว่าหน้าแรกของเว็บจะเป็นอะไร มีสองทางเลือก คือ Your latest posts ที่จะแสดงบทความล่าสุดเป็นหน้าแรก หรือ A static page ที่ให้เลือกหน้า Homepage เฉพาะ ซึ่งเหมาะกับเว็บธุรกิจที่อยากให้หน้าแรกสวยงามค่ะ

**Host A:** ส่วน Discussion Settings คือการตั้งค่าเกี่ยวกับ Comments ครับ สามารถเลือกได้ว่าจะเปิดให้คอมเมนต์ได้หรือไม่ จะต้องอนุมัติก่อนแสดงหรือไม่ และจะแสดงกี่ระดับครับ สำหรับเว็บธุรกิจหลายแห่งเลือกปิด Comments เลยเพื่อลดภาระในการดูแลและป้องกัน Spam ครับ

**Host B:** ใช่ค่ะ ถ้าเปิด Comments ก็ต้องดูแลเรื่อง Spam ด้วย แนะนำให้ตั้งค่า Comment must be manually approved ไว้ หรือใช้ Plugin อย่าง Akismet ช่วยกรอง Spam Comments ค่ะ

---

## ช่วงที่ 5: Media Settings & Security

**Host A:** เรื่อง Media Settings ก็สำคัญครับ เข้าไปที่ Settings แล้วเลือก Media ตรงนี้จะตั้งค่าขนาดรูปภาพเริ่มต้นได้ 3 ขนาด คือ Thumbnail, Medium และ Large WordPress จะสร้างรูปภาพทั้ง 3 ขนาดนี้อัตโนมัติทุกครั้งที่อัปโหลดรูป ถ้าไม่ต้องการให้สร้างบางขนาดก็ตั้งค่าเป็น 0 ได้ครับ ช่วยประหยัดพื้นที่

**Host B:** สุดท้ายคือเรื่อง Security ค่ะ ซึ่งเป็นเรื่องที่หลายคนมองข้ามจริงๆ การ Hardening เบื้องต้นที่ควรทำ ได้แก่ เปลี่ยน Username จาก admin เป็นชื่ออื่น เพราะ admin เป็นเป้าหมายแรกของ Hacker, ใช้รหัสผ่านที่แข็งแกร่ง, จำกัดจำนวนครั้งที่ Login ผิดได้ และอัปเดต WordPress กับ Plugin ให้เป็นเวอร์ชันล่าสุดเสมอค่ะ

**Host A:** เพิ่มเติมอีกเรื่องครับคือ Privacy Settings ที่เข้าไปตั้งค่าได้จาก Settings แล้วเลือก Privacy เราสามารถสร้างหน้า Privacy Policy ได้จากตรงนี้ ซึ่งเป็นสิ่งที่กฎหมายหลายประเทศกำหนดให้เว็บไซต์ต้องมีครับ โดยเฉพาะถ้าเก็บข้อมูลส่วนบุคคลของผู้ใช้

## สรุปปิดรายการ

**Host B:** สรุปวันนี้นะคะ เราได้เรียนรู้การปรับแต่ง WordPress Admin Panel หลายด้าน ตั้งแต่ Dashboard Widgets, User Roles, Admin Menu, Writing Settings รวมถึง Permalinks, Reading Settings, Discussion Settings, Media Settings, Privacy Settings และ Security Hardening ค่ะ

**Host A:** เรื่องเหล่านี้อาจดูเล็กน้อย แต่ถ้าตั้งค่าให้ถูกต้องตั้งแต่แรก จะช่วยให้การบริหารเว็บไซต์ราบรื่นและปลอดภัยมากครับ อย่าลืมกลับไปตรวจสอบ Settings ของเว็บไซต์ตัวเองด้วยนะครับ

**Host B:** ในตอนถัดไปเราจะมีเรื่องน่าสนใจเกี่ยวกับการสร้างเว็บไซต์อีก อย่าลืมติดตามนะคะ แล้วพบกันค่ะ

**Host A:** ขอบคุณที่รับฟังครับ สวัสดีครับ
