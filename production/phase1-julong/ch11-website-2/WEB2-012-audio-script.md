## เปิดรายการ

**Host A:** สวัสดีครับทุกคน วันนี้เราจะมาพูดถึงหัวข้อที่สำคัญมากสำหรับทุกเว็บไซต์ นั่นก็คือเรื่องความเร็วครับ และเครื่องมือที่จะมาช่วยเรื่องนี้ก็คือ WP-Rocket ปลั๊กอิน Cache ที่ได้รับความนิยมมากที่สุดตัวหนึ่งสำหรับ WordPress

**Host B:** ใช่ค่ะ ต้องบอกว่าความเร็วเว็บไซต์สำคัญมากทั้งในแง่ SEO และประสบการณ์ผู้ใช้ Google บอกชัดเจนเลยว่าความเร็วเป็นหนึ่งในปัจจัยจัดอันดับ ถ้าเว็บช้า ทั้งอันดับหายและลูกค้าหนีหมดค่ะ WP-Rocket จะช่วยแก้ปัญหานี้ได้อย่างมีประสิทธิภาพ

---

## ช่วงที่ 1: File Optimization — ลดขนาดไฟล์

**Host A:** เริ่มจากส่วนแรกกันเลยครับ คือ File Optimization ซึ่งเป็นการจัดการไฟล์ CSS และ JavaScript ให้เล็กลงและโหลดเร็วขึ้น มีหลายตัวเลือกครับ อันแรกคือ Minify CSS ซึ่งจะลบช่องว่าง บรรทัดว่าง และ comment ที่ไม่จำเป็นออกจากไฟล์ CSS ทำให้ไฟล์เล็กลง

**Host B:** จากนั้นก็มี Combine CSS ค่ะ ซึ่งจะรวมไฟล์ CSS หลายๆ ไฟล์เป็นไฟล์เดียว ลดจำนวน HTTP Requests แต่ต้องระวังนะคะ บางครั้งการ Combine อาจทำให้หน้าตาเว็บเพี้ยนได้ ถ้าเจอปัญหาก็ปิดตัวนี้แล้วใช้แค่ Minify ก็พอค่ะ

**Host A:** สำหรับ JavaScript ก็มี Minify JS และ Combine JS เหมือนกันครับ แต่ที่สำคัญมากคือ Defer JS Loading ครับ ตัวนี้จะทำให้ JavaScript ไม่ขวางการโหลดหน้าเว็บ ปกติ browser จะหยุดแสดงหน้าเว็บเพื่อรอดาวน์โหลดและรัน JavaScript ก่อน แต่ Defer จะให้ HTML โหลดเสร็จก่อน แล้วค่อยรัน JS ทีหลัง ทำให้ผู้ใช้เห็นเนื้อหาเร็วขึ้นมากครับ

**Host B:** เคล็ดลับคือเปิดทีละตัวแล้วทดสอบค่ะ อย่าเปิดทุกอย่างพร้อมกัน เพราะถ้ามีปัญหาจะไม่รู้ว่าตัวไหนที่ทำให้เว็บพัง

## ช่วงที่ 2: Media Settings — จัดการสื่อให้เบา

**Host A:** ส่วนที่สองคือ Media Settings ครับ ตัวเด่นเลยคือ LazyLoad ซึ่งจะทำให้รูปภาพ iframes และวิดีโอไม่โหลดทั้งหมดพร้อมกัน แต่จะโหลดเฉพาะตอนที่ผู้ใช้เลื่อนหน้าจอลงมาถึง เว็บที่มีรูปเยอะจะเร็วขึ้นมหาศาลเลยครับ

**Host B:** นอกจากนี้ยังมีตัวเลือก Add Missing Image Dimensions ค่ะ ซึ่งจะเพิ่มค่า width และ height ให้กับรูปภาพที่ไม่มี ช่วยลดปัญหา Layout Shift คือหน้าเว็บกระตุกเมื่อรูปโหลดเสร็จ ซึ่งส่งผลต่อ Core Web Vitals ของ Google ด้วยนะคะ

**Host A:** และถ้าเว็บไซต์ใช้รูปภาพแบบ WebP ก็มีตัวเลือก WebP Compatibility ครับ WebP เป็นรูปแบบภาพจาก Google ที่ขนาดเล็กกว่า JPEG/PNG ประมาณ 25-30% แต่คุณภาพเท่ากัน WP-Rocket จะช่วยส่งไฟล์ WebP ให้ browser ที่รองรับโดยอัตโนมัติครับ

---

## ช่วงที่ 3: Preloading และ Advanced Rules

**Host B:** มาถึงส่วน Preloading ค่ะ ตัวนี้จะช่วยให้ WP-Rocket สร้าง cache ล่วงหน้า โดยไล่ Crawl หน้าเว็บตาม Sitemap ที่เรากำหนด ทำให้ผู้เข้าชมคนแรกไม่ต้องรอสร้าง cache แต่จะได้หน้าที่ cache ไว้แล้วทันทีค่ะ

**Host A:** นอกจากนี้ยังมี DNS Prefetch ครับ ซึ่งจะบอกให้ browser ไป resolve DNS ของ domain ภายนอกล่วงหน้า เช่น ถ้าเว็บเราโหลด font จาก fonts.googleapis.com หรือรูปภาพจาก CDN ก็ใส่ domain เหล่านั้นเข้าไป จะช่วยลดเวลาโหลดได้อีกครับ

**Host B:** ส่วน Advanced Rules เป็นการตั้งค่าขั้นสูงค่ะ เช่น Never Cache URLs สำหรับหน้าที่ไม่ต้องการ cache เช่น หน้า checkout หน้า my account Always Purge URLs สำหรับหน้าที่ต้อง clear cache ทุกครั้งที่มีการอัปเดต และ Cache Query Strings สำหรับ URL ที่มีพารามิเตอร์พิเศษค่ะ

---

## ช่วงที่ 4: Database, CDN และ Heartbeat

**Host A:** มาถึงส่วน Database Optimization ครับ WP-Rocket ช่วยทำความสะอาดฐานข้อมูลได้ด้วย ลบ Post Revisions ที่สะสมจำนวนมาก ลบ Auto Drafts ลบ Transients ลบ Spam Comments และ Optimize Tables เพื่อลดขนาดฐานข้อมูลให้เล็กลง ทำให้ query เร็วขึ้นครับ

**Host B:** ถัดมาคือ CDN Integration ค่ะ CDN หรือ Content Delivery Network คือเครือข่าย server ทั่วโลกที่ช่วยส่งเนื้อหาเว็บไซต์จาก server ที่ใกล้ผู้เข้าชมที่สุด WP-Rocket สามารถเชื่อมต่อกับ CDN ได้ง่าย และยังมี Cloudflare Add-on สำหรับคนที่ใช้ Cloudflare อยู่แล้วค่ะ

**Host A:** สุดท้ายคือ Heartbeat API Control ครับ WordPress Heartbeat API เป็นระบบที่ส่งข้อมูลกลับไปหา server ทุก 15-60 วินาที ซึ่งกินทรัพยากรมาก โดยเฉพาะบนโฮสติ้งที่มีจำกัด WP-Rocket ช่วยลดความถี่หรือปิด Heartbeat API ในหน้าที่ไม่จำเป็นได้ครับ

---

## ช่วงที่ 5: ทดสอบก่อน-หลัง

**Host B:** สุดท้ายนี้สำคัญมากค่ะ คือการทดสอบก่อนและหลังติดตั้ง WP-Rocket ด้วย GTmetrix ค่ะ ก่อนเปิดใช้ WP-Rocket ให้ไปทดสอบเว็บที่ gtmetrix.com บันทึกคะแนนไว้ จากนั้นเปิด WP-Rocket ตั้งค่าต่างๆ แล้วทดสอบอีกครั้ง จะเห็นความแตกต่างชัดเจนเลยค่ะ

**Host A:** โดยทั่วไปหลังติดตั้ง WP-Rocket คะแนน PageSpeed จะเพิ่มขึ้น 20-40 คะแนนครับ และเวลาโหลดหน้าอาจลดลงได้ 40-60% เลยทีเดียว ซึ่งส่งผลดีทั้งต่อ SEO และอัตราการแปลงลูกค้าครับ

## สรุปปิดรายการ

**Host B:** สรุปวันนี้นะคะ WP-Rocket มีฟีเจอร์หลักๆ ได้แก่ File Optimization สำหรับย่อ CSS/JS, Media LazyLoad สำหรับรูป/วิดีโอ, Preloading สำหรับสร้าง cache ล่วงหน้า, Database Cleanup ทำความสะอาดฐานข้อมูล, CDN Integration และ Heartbeat Control อย่าลืมทดสอบด้วย GTmetrix ก่อนและหลังค่ะ

**Host A:** WP-Rocket เป็นการลงทุนที่คุ้มค่ามากสำหรับความเร็วเว็บไซต์ ตั้งค่าครั้งเดียวได้ผลลัพธ์ถาวร ในตอนถัดไป เราจะมาพูดถึงเรื่องการสำรองข้อมูลด้วย UpdraftPlus ซึ่งสำคัญไม่แพ้กัน อย่าพลาดนะครับ

**Host B:** แล้วพบกันตอนหน้าค่ะ สวัสดีค่ะ
