## เฉลย (Answer Key)

| ข้อ | คำตอบ | คำอธิบายสั้น |
|-----|-------|-------------|
| 1 | **B** | เป้าหมายคือโหลดภายใน 3 วินาที ผู้ใช้ 53% จะออกจากเว็บถ้าช้ากว่านี้ |
| 2 | **A** | Google ใช้ Core Web Vitals (LCP, FID, CLS) เป็นปัจจัยในการจัดอันดับจริง |
| 3 | **B** | Browser Caching เก็บไฟล์ในเบราว์เซอร์ผู้ใช้ ครั้งที่ 2 ไม่ต้องโหลดใหม่ |
| 4 | **C** | GZIP Compression บีบอัดไฟล์ได้ 60-80% เหมือนการซิปไฟล์ก่อนส่ง |
| 5 | **B** | รูปภาพคิดเป็น 50-80% ของขนาดหน้าเว็บ เป็นตัวการหลักที่ทำให้ช้า |
| 6 | **B** | Lazy Loading โหลดรูปเฉพาะตอนเลื่อนมาถึง ไม่ได้โหลดทั้งหมดตั้งแต่แรก |
| 7 | **D** | WebP เป็นรูปแบบจาก Google ไฟล์เล็กกว่า JPEG 25-34% คุณภาพเทียบเท่า |
| 8 | **B** | CDN = Content Delivery Network กระจายไฟล์ไปยัง server ทั่วโลก |
| 9 | **Google PageSpeed Insights** | เครื่องมือจาก Google ที่ให้คะแนน 0-100 พร้อมคำแนะนำการปรับปรุง |
| 10 | **A** | Minification เอาช่องว่าง comment ออก ลดขนาดไฟล์ 10-30% โดยทำงานเหมือนเดิม |
