## เปิดรายการ

**Host A:** สวัสดีครับทุกคน วันนี้เราจะมาพูดถึงหัวข้อที่หลายคนมักมองข้าม แต่เป็นสิ่งที่สำคัญที่สุดอย่างหนึ่งสำหรับทุกเว็บไซต์ นั่นก็คือเรื่องการสำรองข้อมูล หรือ Backup ครับ เครื่องมือที่เราจะใช้วันนี้คือ UpdraftPlus ปลั๊กอิน Backup ยอดนิยมอันดับ 1 ของ WordPress

**Host B:** ใช่ค่ะ ต้องบอกว่าเรื่อง Backup นี่ไม่ใช่ "ถ้า" แต่เป็น "เมื่อไร" ค่ะ ไม่ใช่ถ้าเว็บพัง แต่เมื่อไรเว็บจะพัง เพราะปัญหาเกิดขึ้นได้ตลอดเวลา ไม่ว่าจะเป็นการอัปเดตปลั๊กอินที่ทำให้เว็บพัง ถูกแฮก หรือแม้แต่ความผิดพลาดของตัวเราเอง ถ้าไม่มี Backup ก็ต้องเริ่มใหม่ตั้งแต่ศูนย์เลยค่ะ

---

## ช่วงที่ 1: ทำไมต้องสำรองข้อมูล

**Host A:** มาเริ่มจากเหตุผลว่าทำไม Backup ถึงสำคัญครับ หนึ่งคือการอัปเดตที่ผิดพลาด WordPress, ปลั๊กอิน หรือธีม มีการอัปเดตบ่อย และบางครั้งการอัปเดตทำให้เว็บพังได้ ถ้ามี Backup ก็กู้คืนได้ทันทีครับ

**Host B:** สองคือเรื่อง Security ค่ะ ทุกปีมีเว็บ WordPress หลายหมื่นเว็บถูกแฮก ถ้ามี Backup ที่สะอาด ก็กู้คืนเป็นเวอร์ชันก่อนถูกแฮกได้ สามคือความผิดพลาดของมนุษย์ค่ะ บางทีเราลบไฟล์ผิด แก้โค้ดผิด ถ้ามี Backup ก็ไม่ต้องตกใจ

**Host A:** และสี่คือปัญหาจากโฮสติ้งครับ server ล่ม ฮาร์ดดิสก์เสีย หรือโฮสติ้งปิดตัว ถ้า Backup อยู่ที่อื่นที่ไม่ใช่ server เดียวกัน ก็ยังมีข้อมูลครบถ้วน ย้ายโฮสติ้งได้ทันทีครับ

---

## ช่วงที่ 2: ตั้งค่า Backup Schedule

**Host B:** มาดูการตั้งค่า UpdraftPlus กันค่ะ หลังติดตั้งและ Activate ให้ไปที่ Settings > UpdraftPlus Backups จากนั้นกดแท็บ Settings สิ่งแรกที่ต้องตั้งคือ Backup Schedule ค่ะ

**Host A:** UpdraftPlus ให้ตั้งตารางแยกกันระหว่าง Files Backup กับ Database Backup ครับ เหตุผลที่ต้องแยกเพราะ Database เปลี่ยนแปลงบ่อยกว่า ทุกครั้งที่มี comment ใหม่ บทความใหม่ หรือ order ใหม่ ฐานข้อมูลเปลี่ยน แต่ไฟล์ Theme/Plugin ไม่ค่อยเปลี่ยนบ่อย

**Host B:** แนะนำตั้ง Database Backup เป็น Daily ค่ะ และ Files Backup เป็น Weekly แล้วก็ตั้ง Retain ไว้สัก 4-7 ชุด เพื่อให้มีหลาย version ย้อนหลังในกรณีที่ไม่รู้ว่าปัญหาเกิดตั้งแต่เมื่อไร

---

## ช่วงที่ 3: Remote Storage

**Host A:** สิ่งสำคัญมากครับ อย่าเก็บ Backup ไว้ที่เดียวกับเว็บไซต์ ถ้า server พัง ทั้งเว็บและ Backup ก็หายไปด้วย UpdraftPlus รองรับ Remote Storage หลายตัวครับ

**Host B:** ตัวที่แนะนำมากที่สุดคือ Google Drive ค่ะ เพราะฟรีมีพื้นที่ 15 GB และเชื่อมต่อง่ายมาก แค่กดเลือก Google Drive แล้ว Authorize ด้วย Google Account ก็เสร็จค่ะ

**Host A:** นอกจากนี้ยังมี Dropbox, Amazon S3 และ Email ครับ แต่ Email ไม่ค่อยแนะนำเพราะไฟล์ Backup มักจะใหญ่เกินไปสำหรับ Email ส่วน Amazon S3 เหมาะสำหรับเว็บใหญ่ที่ต้องการความน่าเชื่อถือสูงสุดครับ

---

## ช่วงที่ 4: สิ่งที่ Backup และการ Restore

**Host B:** มาดูว่า UpdraftPlus Backup อะไรบ้างค่ะ มี 5 ส่วน ได้แก่ Database ซึ่งคือหัวใจของเว็บ เก็บบทความ หน้า ความคิดเห็น การตั้งค่าทุกอย่าง Plugins คือปลั๊กอินทั้งหมดที่ติดตั้ง Themes คือธีมทั้งหมด Uploads คือไฟล์สื่อทั้งหมด รูปภาพ วิดีโอ PDF และ Others คือไฟล์อื่นๆ ที่ไม่อยู่ในหมวดข้างต้นค่ะ

**Host A:** สำหรับการ Restore หรือกู้คืน UpdraftPlus ทำได้ง่ายมากครับ ไปที่แท็บ Existing Backups จะเห็นรายการ Backup ที่มีทั้งหมด กดปุ่ม Restore ที่ชุดที่ต้องการ เลือกว่าจะ Restore อะไรบ้าง Database, Plugins, Themes, Uploads หรือทั้งหมด แล้วก็กด Restore เลยครับ ระบบจะจัดการให้ทั้งหมด

**Host B:** ข้อดีของ UpdraftPlus คือมันเก็บ Backup ของเดิมไว้ก่อนจะ Restore ด้วยค่ะ ถ้า Restore แล้วมีปัญหา ก็กลับไปเวอร์ชันก่อน Restore ได้ ปลอดภัยมากค่ะ

---

## ช่วงที่ 5: การย้ายเว็บและทดสอบ

**Host A:** อีกฟีเจอร์ที่เจ๋งมากของ UpdraftPlus คือการ Migrate หรือ Clone เว็บไซต์ครับ ถ้าอยากย้ายเว็บไปโฮสติ้งใหม่ หรือสร้างเว็บตัวทดสอบ ก็ใช้ Backup จาก UpdraftPlus ไป Restore บน server ใหม่ได้เลย ในเวอร์ชันฟรีต้องทำด้วยตนเอง แต่เวอร์ชัน Premium มี Migrator tool ที่ทำให้อัตโนมัติครับ

**Host B:** และเรื่องสุดท้ายที่สำคัญมากค่ะ คือ **การทดสอบ Backup** อย่าแค่ตั้งค่า Backup แล้วลืมไป ต้องทดสอบว่า Restore ได้จริงด้วยนะคะ แนะนำทดสอบอย่างน้อย 3-6 เดือนครั้ง สร้าง staging site แล้วลอง Restore ดู ถ้า Restore ไม่ได้ Backup ก็ไม่มีประโยชน์ค่ะ

**Host A:** พูดถึง Free vs Premium ครับ เวอร์ชันฟรีของ UpdraftPlus ทำได้เกือบทุกอย่างที่พูดมา ส่วน Premium เพิ่มเรื่อง Incremental Backup ที่ Backup เฉพาะไฟล์ที่เปลี่ยน Migrator tool และ More Storage Options ถ้าเว็บไม่ใหญ่มาก เวอร์ชันฟรีเพียงพอเลยครับ

## สรุปปิดรายการ

**Host B:** สรุปวันนี้นะคะ UpdraftPlus เป็นปลั๊กอิน Backup อันดับ 1 ของ WordPress ตั้งค่า Schedule แยก Database (Daily) กับ Files (Weekly) เลือก Remote Storage เป็น Google Drive Backup ครอบคลุม 5 ส่วน ได้แก่ Database, Plugins, Themes, Uploads และ Others Restore ง่ายแค่คลิกเดียว และอย่าลืมทดสอบ Backup เป็นประจำค่ะ

**Host A:** Backup คือประกันชีวิตของเว็บไซต์ครับ ไม่ว่าอะไรจะเกิดขึ้น ถ้ามี Backup ก็ไม่ต้องกลัว ตั้งค่าเสร็จแล้วก็นอนหลับสบายได้ หวังว่าซีรีส์ Ch11 ทั้งหมดจะเป็นประโยชน์กับทุกคนนะครับ

**Host B:** แล้วพบกันตอนหน้าค่ะ สวัสดีค่ะ

**Host A:** สวัสดีครับ
